Module 7 : Docker Compose et Orchestration

7.1 Introduction à Docker Compose

Objectifs Pédagogiques

Dans cette section, vous allez apprendre :

  • Comprendre le rôle de Docker Compose
  • Les avantages de l'orchestration de conteneurs
  • La structure des fichiers docker-compose.yml
  • Les cas d'usage typiques

Qu'est-ce que Docker Compose ?

Docker Compose est un outil qui permet de définir et de gérer des applications multi-conteneurs. Il utilise un fichier YAML pour configurer les services, réseaux et volumes de l'application.

  • Définition déclarative des services
  • Gestion simplifiée des dépendances
  • Configuration centralisée
  • Environnements de développement reproductibles

Structure de Base

version: '3.8'

services:
  web:
    image: nginx:latest
    ports:
      - "80:80"
  
  db:
    image: mysql:8.0
    environment:
      - MYSQL_ROOT_PASSWORD=secret
      
volumes:
  db_data:

networks:
  app_network:

Points Clés

  • Configuration : Format YAML simple et lisible
  • Services : Définition des conteneurs et leurs relations
  • Environnement : Gestion des variables et des dépendances
  • Scalabilité : Facilité de mise à l'échelle

7.2 Configuration des Services

Objectifs Pédagogiques

Dans cette section, vous allez apprendre :

  • Définir des services dans docker-compose.yml
  • Configurer les options des services
  • Gérer les dépendances entre services
  • Utiliser les variables d'environnement

Options de Configuration

services:
  web:
    build: ./web
    image: mon-app:latest
    container_name: web_app
    ports:
      - "8080:80"
    volumes:
      - ./src:/app
    environment:
      - DEBUG=1
    depends_on:
      - db
    restart: always
    networks:
      - frontend
      - backend

Gestion des Dépendances

services:
  web:
    depends_on:
      db:
        condition: service_healthy
      redis:
        condition: service_started
  
  db:
    healthcheck:
      test: ["CMD", "mysqladmin", "ping", "-h", "localhost"]
      interval: 10s
      timeout: 5s
      retries: 5

Points Clés

  • Options : Configuration détaillée des services
  • Dépendances : Gestion de l'ordre de démarrage
  • Environnement : Variables et configurations
  • Santé : Surveillance des services

7.3 Gestion des Volumes et Réseaux

Objectifs Pédagogiques

Dans cette section, vous allez apprendre :

  • Configurer les volumes dans Docker Compose
  • Gérer les réseaux entre services
  • Utiliser les drivers de stockage
  • Optimiser la communication inter-services

Configuration des Volumes

volumes:
  db_data:
    driver: local
    driver_opts:
      type: none
      device: /data/mysql
      o: bind
  
services:
  db:
    volumes:
      - db_data:/var/lib/mysql
      - ./init:/docker-entrypoint-initdb.d
      - type: tmpfs
        target: /tmp

Configuration des Réseaux

networks:
  frontend:
    driver: bridge
    ipam:
      driver: default
      config:
        - subnet: 172.28.0.0/16
  
  backend:
    internal: true
    driver: overlay
    
services:
  web:
    networks:
      frontend:
        ipv4_address: 172.28.1.1
      backend:
        aliases:
          - webapp

Points Clés

  • Volumes : Persistance et partage des données
  • Réseaux : Isolation et communication
  • Drivers : Flexibilité du stockage
  • Performance : Optimisation des ressources

7.4 Commandes Docker Compose

Objectifs Pédagogiques

Dans cette section, vous allez apprendre :

  • Utiliser les commandes de base
  • Gérer le cycle de vie des services
  • Surveiller les conteneurs
  • Déboguer les problèmes courants

Commandes de Base

# Démarrer les services
docker compose up -d

# Arrêter les services
docker compose down

# Voir les logs
docker compose logs -f

# Lister les services
docker compose ps

# Exécuter une commande
docker compose exec web npm test

Gestion des Services

# Reconstruire les images
docker compose build

# Mettre à jour un service
docker compose up -d --no-deps --build web

# Scaler un service
docker compose up -d --scale web=3

# Voir les processus
docker compose top

Points Clés

  • Gestion : Commandes simples et efficaces
  • Surveillance : Outils de monitoring intégrés
  • Maintenance : Facilité de mise à jour
  • Débogage : Accès aux logs et diagnostics

7.5 Environnements Multiples

Objectifs Pédagogiques

Dans cette section, vous allez apprendre :

  • Gérer plusieurs environnements
  • Utiliser les fichiers .env
  • Surcharger les configurations
  • Organiser les fichiers de configuration

Structure des Fichiers

# docker-compose.yml (base)
version: '3.8'
services:
  web:
    image: nginx:latest

# docker-compose.override.yml (dev)
services:
  web:
    volumes:
      - ./src:/app
    environment:
      - DEBUG=1

# docker-compose.prod.yml
services:
  web:
    restart: always
    environment:
      - DEBUG=0

Variables d'Environnement

# .env
DB_HOST=localhost
DB_USER=root
DB_PASS=secret

# .env.prod
DB_HOST=db.production
DB_USER=${PROD_DB_USER}
DB_PASS=${PROD_DB_PASS}

# Utilisation
docker compose --env-file .env.prod up -d

Points Clés

  • Configuration : Séparation des environnements
  • Variables : Gestion flexible des paramètres
  • Sécurité : Protection des données sensibles
  • Maintenance : Organisation claire des fichiers

7.6 Travail Pratique : Application Multi-Services

Objectifs du TP

Dans ce travail pratique, vous allez :

  • Créer une application complète avec plusieurs services
  • Configurer les environnements de développement et production
  • Gérer les données persistantes
  • Mettre en place un système de monitoring

Structure du Projet

project/
  ├── docker-compose.yml
  ├── docker-compose.override.yml
  ├── docker-compose.prod.yml
  ├── .env
  ├── .env.prod
  ├── web/
  │   ├── Dockerfile
  │   └── src/
  ├── api/
  │   ├── Dockerfile
  │   └── src/
  └── db/
      └── init/

Configuration Complète

version: '3.8'

services:
  web:
    build: ./web
    ports:
      - "80:80"
    depends_on:
      - api
    networks:
      - frontend
  
  api:
    build: ./api
    depends_on:
      db:
        condition: service_healthy
    networks:
      - frontend
      - backend
    environment:
      - DB_HOST=db
      - DB_USER=${DB_USER}
      - DB_PASS=${DB_PASS}
  
  db:
    image: postgres:13
    volumes:
      - db_data:/var/lib/postgresql/data
      - ./db/init:/docker-entrypoint-initdb.d
    networks:
      - backend
    healthcheck:
      test: ["CMD", "pg_isready"]
      interval: 10s
      timeout: 5s
      retries: 5
  
  monitoring:
    image: grafana/grafana
    ports:
      - "3000:3000"
    volumes:
      - grafana_data:/var/lib/grafana
    networks:
      - monitoring

volumes:
  db_data:
  grafana_data:

networks:
  frontend:
  backend:
    internal: true
  monitoring:

Commandes de Déploiement

# Développement
docker compose up -d

# Production
docker compose -f docker-compose.yml \
  -f docker-compose.prod.yml \
  --env-file .env.prod \
  up -d

# Vérification
docker compose ps
docker compose logs -f

Points Clés du TP

  • Architecture : Structure multi-services cohérente
  • Configuration : Gestion des environnements
  • Données : Persistance et sécurité
  • Monitoring : Surveillance de l'application
