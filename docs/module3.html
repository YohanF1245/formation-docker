3.1 Quelle est la différence entre un conteneur Docker et une image Docker ?

Prérequis

Pour saisir pleinement les concepts présentés dans ce chapitre, il est recommandé d'avoir :

  • Une compréhension basique de Docker et de son rôle dans le développement et le déploiement d'applications
  • Une familiarité avec les principes de virtualisation et d'isolation dans l'environnement de développement

Objectifs Pédagogiques

Après avoir complété ce chapitre, vous devriez être capable de :

  • Expliquer la différence fondamentale entre une image Docker et un conteneur Docker
  • Comprendre le concept d'instance tel qu'il s'applique aux conteneurs Docker

3.1.1 Instance

Imaginez une chanson sur une plateforme de streaming comme Spotify. Chaque fois que vous jouez cette chanson, vous créez une "instance" de cette chanson. Même si la chanson originale est la même pour tout le monde, chaque écoute est une instance unique pour l'auditeur.

3.1.2 Image Docker

Une image Docker est comparable à la chanson enregistrée sur Spotify. C'est une sorte de "blueprint" ou de recette qui contient toutes les instructions et les fichiers nécessaires pour créer un conteneur Docker. L'image est immuable, c'est-à-dire qu'elle ne change pas, mais elle peut être transférée (téléchargée ou partagée), modifiée (par création d'une nouvelle image), ou supprimée selon les besoins.

3.1.3 Conteneur Docker

Un conteneur Docker, en revanche, est comme la chanson que vous jouez actuellement sur votre appareil. C'est une instance en temps réel de l'image Docker. Chaque conteneur est une application ou un environnement isolé et exécutable qui fonctionne à partir de l'image mais dans son propre espace indépendant.

Résumé

  • Image Docker : Agit comme un modèle ou une recette immuable qui contient toutes les instructions et fichiers nécessaires pour créer un conteneur Docker. Elle est comparable à une chanson sur une plateforme de streaming, existant en tant que blueprint prêt à être exécuté.
  • Conteneur Docker : Représente une instance exécutable de l'image Docker, fonctionnant dans un environnement isolé. Il est analogue à jouer la chanson sur un appareil, où chaque lecture crée une expérience unique bien que basée sur le même modèle original.

3.2 Les Registres Docker et Docker Hub

Prérequis

Avant d'explorer ce chapitre, il est conseillé d'avoir :

  • Une compréhension de base de Docker et de la conteneurisation
  • Une familiarité avec les concepts de base de la gestion des images Docker

Objectifs Pédagogiques

À la fin de ce chapitre, il est attendu de comprendre :

  • Le rôle et l'importance des registres Docker pour le stockage et la distribution d'images Docker
  • Les fonctionnalités et les avantages de Docker Hub en tant que registre Docker public
  • Comment utiliser les tags pour gérer différentes versions d'images Docker
  • Les étapes pour lier un terminal au compte Docker Hub et les opérations de base pour pousser et tirer des images
  • Les alternatives à Docker Hub et les options d'abonnement disponibles

3.2.1 Qu'est-ce qu'un Registre Docker ?

Un registre Docker est un service de stockage et de distribution d'images Docker. Il est similaire à une bibliothèque où les images Docker sont les livres, disponibles pour être empruntés (téléchargés) et utilisés.

3.2.2 Pourquoi Utiliser un Registre Docker ?

Les registres Docker, comme Docker Hub, offrent un espace centralisé pour stocker et partager des images Docker. Cela facilite le partage d'applications et d'environnements entre différentes équipes ou individus.

3.2.3 Docker Hub : Le Registre Docker le Plus Populaire

Docker Hub est le registre public le plus connu et utilisé pour les images Docker. Il propose des images officielles maintenues par des équipes de développement ainsi que des images créées par la communauté.

3.2.3.1 Création de Compte et Connexion

Pour utiliser Docker Hub, il est nécessaire de créer un compte. Une fois inscrit, vous pouvez télécharger (pull) des images publiques et télécharger (push) vos propres images.

3.2.3.2 Niveaux d'Abonnement

  • Gratuit : Permet de télécharger des images publiques et de télécharger un nombre limité d'images privées
  • Payant : Offre des options avancées, notamment un nombre plus élevé d'images privées et d'autres fonctionnalités

3.2.4 Tags dans Docker Hub

Les tags dans Docker sont utilisés pour spécifier des versions ou des configurations différentes d'une même image.

3.2.4.1 Comment Taguer une Image ?

Pour taguer une image, utilisez la commande docker tag. Par exemple :

docker tag mon_apache mon_utilisateur/mon_apache:version1

Cela crée un tag version1 pour l'image mon_apache.

3.2.4.2 Gestion des Tags sur Docker Hub

Sur Docker Hub, les tags aident à organiser et à identifier différentes versions d'une image. Vous pouvez avoir plusieurs tags pour une même image, facilitant ainsi le suivi des versions.

3.2.5 Lier le Terminal au Compte Docker Hub

Pour lier votre terminal à votre compte Docker Hub, vous devez vous connecter via la commande docker login. Cela permet de pousser et de tirer des images depuis votre compte Docker Hub.

3.2.6 Exemples de Registres Docker Connus

  • Docker Hub
  • Google Container Registry
  • Amazon Elastic Container Registry

Résumé

  • Registres Docker : Services de stockage et de distribution pour les images Docker, agissant comme des bibliothèques centralisées qui facilitent le partage et l'utilisation des images entre équipes ou individus.
  • Docker Hub : Le registre Docker public le plus utilisé, offrant des images officielles et communautaires. Il nécessite la création d'un compte pour pousser (push) et tirer (pull) des images.
  • Tags : Utilisés pour spécifier des versions ou configurations d'images Docker, permettant une gestion efficace des différentes versions d'une même image.
  • Lier le Terminal à Docker Hub : Nécessite l'utilisation de docker login pour authentifier et permettre le transfert d'images vers et depuis Docker Hub.
  • Alternatives à Docker Hub : Incluent Google Container Registry, Amazon Elastic Container Registry, et d'autres, offrant des services similaires avec des intégrations spécifiques à leurs écosystèmes cloud.

3.3 Gestion des Images Docker

Prérequis

Pour suivre cette section efficacement, assurez-vous d'avoir :

  • Docker installé et fonctionnel sur votre système
  • Une connexion Internet pour télécharger les images
  • Les notions de base sur les images Docker et Docker Hub

Objectifs Pédagogiques

À la fin de ce chapitre, vous serez capable de :

  • Rechercher et télécharger des images depuis Docker Hub
  • Gérer le cycle de vie des images Docker
  • Comprendre et utiliser les commandes essentielles de gestion d'images
  • Optimiser l'utilisation de l'espace disque avec les images

3.3.1 Rechercher des Images

La recherche d'images peut se faire soit via l'interface web de Docker Hub, soit en ligne de commande.

docker search nginx

Cette commande affiche une liste d'images correspondant au terme "nginx", avec des informations comme le nombre d'étoiles et le statut officiel.

3.3.2 Télécharger des Images

Pour télécharger une image, utilisez la commande docker pull.

Télécharger la dernière version :

docker pull nginx:latest

Télécharger une version spécifique :

docker pull nginx:1.24

3.3.3 Lister les Images

Pour voir les images disponibles localement :

docker images

Cette commande affiche les informations suivantes pour chaque image :

  • REPOSITORY : nom de l'image
  • TAG : version de l'image
  • IMAGE ID : identifiant unique de l'image
  • CREATED : date de création
  • SIZE : taille de l'image

3.3.4 Supprimer des Images

Pour libérer de l'espace, vous pouvez supprimer les images inutilisées.

Supprimer une image spécifique :

docker rmi nginx:latest

Supprimer toutes les images inutilisées :

docker image prune

3.3.5 Inspecter une Image

Pour obtenir des informations détaillées sur une image :

docker image inspect nginx:latest

Cette commande affiche des informations détaillées comme les couches de l'image, les variables d'environnement, l'architecture, etc.

Résumé

  • Recherche d'Images : Utilisez docker search ou l'interface web de Docker Hub pour trouver les images dont vous avez besoin.
  • Téléchargement : La commande docker pull permet de télécharger des images depuis un registre.
  • Gestion Locale : Utilisez docker images pour lister et docker rmi pour supprimer les images.
  • Inspection : La commande docker image inspect fournit des informations détaillées sur une image.
  • Maintenance : Utilisez régulièrement docker image prune pour nettoyer les images inutilisées.

3.4 Gestion des Conteneurs Docker

Prérequis

Pour cette section, vous devez avoir :

  • Une compréhension des images Docker
  • Docker installé et configuré sur votre système
  • Au moins une image Docker téléchargée localement

Objectifs Pédagogiques

À la fin de ce chapitre, vous serez capable de :

  • Créer et démarrer des conteneurs Docker
  • Gérer le cycle de vie des conteneurs
  • Comprendre et utiliser les options de configuration des conteneurs
  • Surveiller l'état et les ressources des conteneurs

3.4.1 Créer et Démarrer des Conteneurs

La commande principale pour créer et démarrer un conteneur est docker run.

Démarrer un conteneur simple :

docker run nginx

Démarrer en mode détaché (background) :

docker run -d nginx

Démarrer avec un nom personnalisé et un port mappé :

docker run -d --name mon_nginx -p 8080:80 nginx

3.4.2 Options de Configuration Courantes

  • -d : Mode détaché (background)
  • -p : Mapping de ports (hôte:conteneur)
  • -v : Montage de volumes
  • -e : Variables d'environnement
  • --name : Nom personnalisé
  • --network : Réseau Docker
  • --restart : Politique de redémarrage

3.4.3 Gérer les Conteneurs en Cours d'Exécution

Lister les conteneurs actifs :

docker ps

Lister tous les conteneurs (y compris arrêtés) :

docker ps -a

Arrêter un conteneur :

docker stop mon_nginx

Redémarrer un conteneur :

docker restart mon_nginx

3.4.4 Supprimer des Conteneurs

Supprimer un conteneur arrêté :

docker rm mon_nginx

Forcer la suppression d'un conteneur en cours d'exécution :

docker rm -f mon_nginx

Nettoyer tous les conteneurs arrêtés :

docker container prune

3.4.5 Inspecter et Surveiller les Conteneurs

Voir les logs d'un conteneur :

docker logs mon_nginx

Suivre les logs en temps réel :

docker logs -f mon_nginx

Voir les statistiques d'utilisation des ressources :

docker stats mon_nginx

Inspecter la configuration d'un conteneur :

docker inspect mon_nginx

Résumé

  • Création et Démarrage : Utilisez docker run avec différentes options pour créer et démarrer des conteneurs selon vos besoins.
  • Gestion du Cycle de Vie : Les commandes docker start, docker stop, et docker restart permettent de contrôler l'état des conteneurs.
  • Surveillance : Utilisez docker logs et docker stats pour surveiller l'activité et les performances des conteneurs.
  • Nettoyage : Les commandes docker rm et docker container prune permettent de gérer efficacement l'espace disque.

3.5 Exécuter des Commandes dans les Conteneurs

Prérequis

Pour cette section, assurez-vous d'avoir :

  • Des conteneurs Docker en cours d'exécution
  • Une compréhension de base des commandes Linux/Unix
  • Les connaissances acquises dans les sections précédentes

Objectifs Pédagogiques

À la fin de ce chapitre, vous serez capable de :

  • Exécuter des commandes dans un conteneur en cours d'exécution
  • Comprendre la différence entre les commandes exec et run
  • Utiliser le mode interactif pour interagir avec les conteneurs
  • Copier des fichiers entre l'hôte et les conteneurs

3.5.1 La Commande docker exec

La commande docker exec permet d'exécuter une commande dans un conteneur en cours d'exécution.

Exécuter une commande simple :

docker exec mon_nginx ls /usr/share/nginx/html

Ouvrir un shell interactif :

docker exec -it mon_nginx /bin/bash

3.5.2 Options Importantes de docker exec

  • -i : Mode interactif (garde STDIN ouvert)
  • -t : Alloue un pseudo-TTY
  • -w : Répertoire de travail dans le conteneur
  • -e : Variables d'environnement
  • -u : Utilisateur à utiliser

3.5.3 Copier des Fichiers

La commande docker cp permet de copier des fichiers entre l'hôte et les conteneurs.

Copier de l'hôte vers le conteneur :

docker cp ./mon_fichier.html mon_nginx:/usr/share/nginx/html/

Copier du conteneur vers l'hôte :

docker cp mon_nginx:/var/log/nginx/access.log ./logs/

3.5.4 Différence entre exec et run

docker exec exécute une commande dans un conteneur existant et en cours d'exécution.

docker run crée et démarre un nouveau conteneur à partir d'une image.

3.5.5 Bonnes Pratiques

  • Utilisez toujours -it pour les sessions interactives
  • Préférez les variables d'environnement aux modifications directes des fichiers
  • Évitez d'installer des packages permanents via exec
  • Documentez les commandes exec fréquemment utilisées

Résumé

  • docker exec : Permet d'exécuter des commandes dans des conteneurs en cours d'exécution, idéal pour le débogage et la maintenance.
  • Mode Interactif : L'option -it permet d'obtenir un shell interactif dans le conteneur.
  • Transfert de Fichiers : docker cp facilite le transfert de fichiers entre l'hôte et les conteneurs.
  • Bonnes Pratiques : Privilégiez les solutions non intrusives et documentez vos interventions dans les conteneurs.

3.6 Monitorer et Dépanner les Conteneurs Docker

Prérequis

Pour cette section, vous devez avoir :

  • Des conteneurs Docker en cours d'exécution
  • Une compréhension des commandes de base Docker
  • Des notions de base en monitoring système

Objectifs Pédagogiques

À la fin de ce chapitre, vous serez capable de :

  • Surveiller les performances des conteneurs
  • Diagnostiquer les problèmes courants
  • Utiliser les outils de débogage Docker
  • Mettre en place des bonnes pratiques de monitoring

3.6.1 Surveiller les Performances

Docker fournit plusieurs commandes pour surveiller les performances des conteneurs.

Voir les statistiques en temps réel :

docker stats

Voir les processus en cours d'exécution :

docker top mon_conteneur

3.6.2 Analyser les Logs

Les logs sont essentiels pour comprendre le comportement des conteneurs.

Voir les derniers logs :

docker logs --tail 50 mon_conteneur

Suivre les logs en temps réel avec timestamp :

docker logs -f --timestamps mon_conteneur

3.6.3 Problèmes Courants et Solutions

Conteneur qui ne démarre pas

  • Vérifier les logs : docker logs mon_conteneur
  • Inspecter la configuration : docker inspect mon_conteneur
  • Vérifier les ports utilisés : docker port mon_conteneur

Problèmes de ressources

  • Surveiller l'utilisation CPU/mémoire : docker stats
  • Vérifier les limites de ressources : docker inspect
  • Nettoyer les ressources inutilisées : docker system prune

3.6.4 Outils de Débogage Avancés

Inspecter les événements Docker :

docker events --filter 'container=mon_conteneur'

Voir l'utilisation des ressources système :

docker system df

3.6.5 Bonnes Pratiques de Monitoring

  • Mettre en place des limites de ressources pour les conteneurs
  • Configurer la rotation des logs
  • Utiliser des outils de monitoring dédiés (ex: Prometheus, Grafana)
  • Mettre en place des alertes sur les métriques critiques
  • Documenter les procédures de dépannage

Résumé

  • Monitoring : Utilisez docker stats et docker top pour surveiller les performances en temps réel.
  • Logs : Les commandes docker logs sont essentielles pour le diagnostic des problèmes.
  • Dépannage : Une approche systématique utilisant les outils Docker permet de résoudre la plupart des problèmes.
  • Bonnes Pratiques : La mise en place de limites de ressources et d'outils de monitoring est cruciale pour la production.

3.7 Travail Pratique : Déployer une Application Web

Objectifs du TP

Dans ce travail pratique, vous allez :

  • Créer et configurer un conteneur nginx
  • Personnaliser le contenu web
  • Gérer les logs et le monitoring
  • Mettre en pratique les commandes apprises

3.7.1 Préparation

Créer un répertoire de travail :

mkdir mon_site_web cd mon_site_web

Créer un fichier HTML simple (index.html) :

echo ' Mon Site Docker

Bienvenue sur mon site hébergé avec Docker !

Cette page est servie depuis un conteneur nginx.

' > index.html

3.7.2 Déploiement du Conteneur

Lancer le conteneur nginx avec le volume monté :

docker run -d --name mon_site -p 8080:80 -v ${PWD}:/usr/share/nginx/html nginx

Cette commande :

  • Lance nginx en arrière-plan (-d)
  • Nomme le conteneur "mon_site"
  • Mappe le port 8080 de l'hôte au port 80 du conteneur
  • Monte le répertoire courant dans le conteneur

3.7.3 Vérification et Tests

Vérifier que le conteneur est en cours d'exécution :

docker ps

Vérifier les logs du conteneur :

docker logs mon_site

Accéder au site :

Ouvrez votre navigateur et accédez à http://localhost:8080

3.7.4 Exercices Pratiques

Exercice 1 : Personnalisation

  • Modifiez le fichier index.html pour ajouter du contenu personnalisé
  • Vérifiez que les changements sont visibles dans le navigateur

Exercice 2 : Monitoring

  • Surveillez les logs d'accès en temps réel
  • Vérifiez l'utilisation des ressources du conteneur

Exercice 3 : Maintenance

  • Arrêtez et redémarrez le conteneur
  • Vérifiez que le site reste accessible
  • Inspectez la configuration du conteneur

3.7.5 Nettoyage

Une fois terminé, nettoyez votre environnement :

docker stop mon_site docker rm mon_site

Points Clés du TP

  • Configuration : Création d'un environnement web simple avec nginx et Docker.
  • Volumes : Utilisation des volumes pour persister et modifier le contenu web.
  • Monitoring : Mise en pratique des commandes de surveillance et de logs.
  • Maintenance : Application des commandes de gestion du cycle de vie des conteneurs.
