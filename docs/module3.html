Module 3 - Conteneurs et Images Docker

Objectifs du module

  • Comprendre la différence entre conteneurs et images
  • Maîtriser la gestion des images Docker
  • Apprendre à manipuler les conteneurs
  • Découvrir Docker Hub et le partage d'images

Les Images Docker

Une image Docker est un modèle en lecture seule qui contient :

  • Un système de fichiers en couches
  • Des métadonnées d'exécution
  • Des dépendances
  • Des configurations

Commandes principales pour les images

docker pull image:tag

Télécharge une image depuis Docker Hub

docker images

Liste les images disponibles localement

docker rmi image

Supprime une image locale

docker build -t nom .

Construit une image depuis un Dockerfile

Les Conteneurs Docker

Un conteneur est une instance exécutable d'une image. Il peut être :

  • Créé, démarré, arrêté, déplacé ou supprimé
  • Connecté à un ou plusieurs réseaux
  • Attaché à du stockage
  • Créé à partir de l'état actuel

Commandes principales pour les conteneurs

docker run image

Crée et démarre un conteneur

docker ps

Liste les conteneurs en cours d'exécution

docker stop container

Arrête un conteneur en cours d'exécution

docker rm container

Supprime un conteneur arrêté

Options courantes de docker run

Option Description Exemple
-d Mode détaché (background) docker run -d nginx
-p Mapping de ports docker run -p 8080:80 nginx
-v Montage de volume docker run -v /host:/container nginx
--name Nom personnalisé docker run --name web nginx

Exercice pratique

Créez un conteneur nginx et exposez-le sur le port 8080 :

# Téléchargement de l'image nginx

docker pull nginx

# Lancement du conteneur

docker run -d -p 8080:80 --name mon-nginx nginx

# Vérification

docker ps
