Module 6 : Réseautage dans Docker

6.1 Introduction au Réseautage Docker

Objectifs Pédagogiques

Dans cette section, vous allez apprendre :

  • Les concepts fondamentaux du réseautage Docker
  • Les différents types de réseaux disponibles
  • Comment les conteneurs communiquent entre eux
  • Les bonnes pratiques en matière de réseautage

Architecture Réseau de Docker

Docker utilise une architecture réseau flexible qui permet aux conteneurs de communiquer entre eux et avec le monde extérieur. Les principaux composants sont :

  • Le réseau bridge par défaut
  • Les réseaux définis par l'utilisateur
  • Les drivers réseau
  • Les interfaces réseau des conteneurs

Types de Réseaux Docker

Type Description Cas d'Usage
bridge Réseau isolé sur l'hôte Applications sur un seul hôte
host Utilise le réseau de l'hôte Performance maximale
none Pas de connectivité réseau Isolation complète
overlay Réseau multi-hôtes Applications distribuées

Points Clés

  • Architecture : Flexible et modulaire
  • Types : Plusieurs options selon les besoins
  • Isolation : Sécurité et séparation des préoccupations
  • Performance : Options optimisées selon les cas d'usage

6.2 Le Réseau Bridge par Défaut

Objectifs Pédagogiques

Dans cette section, vous allez apprendre :

  • Le fonctionnement du réseau bridge par défaut
  • Comment les conteneurs communiquent sur ce réseau
  • Les limitations du réseau bridge par défaut
  • Les cas d'usage appropriés

Caractéristiques du Bridge par Défaut

# Inspecter le réseau bridge par défaut
docker network inspect bridge

# Lister les réseaux
docker network ls

Communication entre Conteneurs

# Créer deux conteneurs sur le réseau bridge
docker run -d --name web nginx
docker run -d --name db mysql

# Tester la connectivité
docker exec web ping db

Limitations

  • Pas de résolution DNS automatique entre conteneurs
  • Configuration manuelle des liens nécessaire
  • Sécurité limitée
  • Pas de contrôle fin sur l'isolation

Points Clés

  • Configuration : Automatique mais basique
  • Usage : Développement et tests simples
  • Limitations : Fonctionnalités restreintes
  • Alternative : Préférer les réseaux définis par l'utilisateur

6.3 Les Réseaux Définis par l'Utilisateur

Objectifs Pédagogiques

Dans cette section, vous allez apprendre :

  • Créer et gérer des réseaux personnalisés
  • Connecter des conteneurs à des réseaux spécifiques
  • Configurer la résolution DNS automatique
  • Isoler les applications avec des réseaux dédiés

Création et Gestion

# Créer un réseau personnalisé
docker network create mon-reseau

# Créer un réseau avec des options spécifiques
docker network create --driver bridge \
  --subnet=172.18.0.0/16 \
  --gateway=172.18.0.1 \
  mon-reseau-custom

Utilisation avec les Conteneurs

# Lancer un conteneur sur un réseau spécifique
docker run -d --name web --network mon-reseau nginx

# Connecter un conteneur existant
docker network connect mon-reseau db

# Déconnecter un conteneur
docker network disconnect mon-reseau db

Avantages

  • Résolution DNS automatique entre conteneurs
  • Isolation améliorée
  • Configuration réseau flexible
  • Meilleure sécurité

Points Clés

  • Flexibilité : Configuration personnalisée selon les besoins
  • Isolation : Séparation claire des environnements
  • DNS : Résolution automatique des noms
  • Sécurité : Contrôle fin des communications

6.4 Configuration des Ports et Exposition des Services

Objectifs Pédagogiques

Dans cette section, vous allez apprendre :

  • Exposer des ports de conteneurs
  • Configurer le mapping de ports
  • Gérer l'accès aux services
  • Sécuriser l'exposition des services

Exposition des Ports

# Exposition automatique
docker run -d -P nginx

# Mapping de port spécifique
docker run -d -p 8080:80 nginx

# Mapping sur une interface spécifique
docker run -d -p 127.0.0.1:8080:80 nginx

Gestion des Ports

# Voir les ports mappés
docker port conteneur-name

# Lister les conteneurs avec leurs ports
docker ps

Bonnes Pratiques

  • Exposer uniquement les ports nécessaires
  • Utiliser des plages de ports cohérentes
  • Documenter les mappings de ports
  • Considérer la sécurité réseau

Points Clés

  • Exposition : Contrôle fin des ports accessibles
  • Mapping : Flexibilité dans la configuration
  • Sécurité : Limitation de l'exposition
  • Documentation : Importance du suivi des ports

6.5 Réseaux Overlay et Communication Multi-Hôtes

Objectifs Pédagogiques

Dans cette section, vous allez apprendre :

  • Comprendre les réseaux overlay
  • Configurer la communication multi-hôtes
  • Gérer les services distribués
  • Sécuriser les communications entre hôtes

Création de Réseaux Overlay

# Initialiser Swarm mode
docker swarm init

# Créer un réseau overlay
docker network create --driver overlay \
  --attachable \
  mon-reseau-overlay

Utilisation avec les Services

# Créer un service sur le réseau overlay
docker service create \
  --name mon-service \
  --network mon-reseau-overlay \
  nginx

Sécurité et Encryption

  • Encryption des données en transit
  • Isolation des réseaux overlay
  • Contrôle d'accès aux services
  • Gestion des certificats TLS

Points Clés

  • Overlay : Communication transparente multi-hôtes
  • Services : Déploiement distribué facilité
  • Sécurité : Communications chiffrées
  • Scalabilité : Architecture distribuée flexible

6.6 Travail Pratique : Configuration d'une Application Multi-Conteneurs

Objectifs du TP

Dans ce travail pratique, vous allez :

  • Créer une application web avec base de données
  • Configurer les réseaux pour la communication
  • Exposer les services nécessaires
  • Tester et valider la configuration

Étape 1 : Préparation

# Créer le réseau pour l'application
docker network create app-network

# Vérifier la création
docker network ls

Étape 2 : Configuration de la Base de Données

# Lancer MySQL
docker run -d \
  --name db \
  --network app-network \
  -e MYSQL_ROOT_PASSWORD=secret \
  -e MYSQL_DATABASE=app \
  mysql:8.0

Étape 3 : Configuration de l'Application Web

# Lancer l'application web
docker run -d \
  --name web \
  --network app-network \
  -p 8080:80 \
  -e DB_HOST=db \
  -e DB_PASSWORD=secret \
  mon-app:latest

Étape 4 : Tests et Validation

# Vérifier les conteneurs
docker ps

# Tester la connexion
docker exec web ping db

# Vérifier les logs
docker logs web

Points Clés du TP

  • Réseaux : Configuration correcte pour la communication
  • Services : Exposition appropriée des ports
  • Sécurité : Isolation des composants
  • Tests : Validation du fonctionnement
